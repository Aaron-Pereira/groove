Help

Quick reference for every groove feature.

▶ Queue (home page)

Adding a single track or album

  • Type Artist - Song Title or Artist - Album Name in the Add form, choose Track or Album, hit Enter.
  • Paste a YouTube URL directly to download from a specific video.

Status meanings

  • pending – waiting in queue.
  • searching – groove is finding the best YouTube match.
  • downloading – yt-dlp is fetching the audio (MP3 192 kbps).
  • tagging – beets is looking up MusicBrainz and filing into your library.
  • done – file is in library/Artist/Album (Year)/.
  • failed – something went wrong. Click Retry, or supply a manual YouTube URL.
▶ Bulk Add

Supported formats

  • Plain text – one entry per line: Artist - Album, Artist - Song, or a URL.
  • Spotify / Exportify CSV – export at exportify.net, upload here.
  • Generic CSV – any CSV with artist/title/album columns.
  • YouTube playlist URL – groove expands it to individual tracks.
  • Artist discography – type artist name, pick albums from MusicBrainz, or use Queue all after the confirmation prompt.

All bulk requests are priority: low. A preview step shows what's new vs. already in library.

▶ Discoveries

Chart sources

  • Billboard Hot 100 – scraped daily.
  • UK Official Top 40 – scraped daily.
  • Last.fm global top tracks – scraped daily.
  • Last.fm genre charts (rock, hip-hop, electronic, and any genres you add in groove.toml) – daily.
  • MusicBrainz new releases for your watchlist artists – weekly.

Auto-queue

Tracks appearing on 2 or more charts this week are auto-queued at low priority. Change the threshold in groove.toml under [auto_queue].

▶ Watchlist
  • Artists you add here are checked weekly against MusicBrainz for new albums.
  • Enable Auto-download new albums to have groove queue new releases automatically.
  • Without that toggle, new releases still appear in Discoveries for manual queuing.
▶ Library
  • Browse all your music by artist, album, title, or year.
  • Click Edit on any track to update tags (calls beet modify under the hood).
  • After editing tags in an external tool, run groove metadata rescan to re-sync beets.

Fix album track numbers / re-tag one album

When an album has wrong track numbers, wrong titles, or every file has the same name, re-import that album folder against MusicBrainz. Beets matches by audio fingerprint and rewrites tags + filenames.

From the groove/ project directory:

uv run beet --config ../groove-data/beets.yaml import -C -I \
  "../groove-data/library/Artist Name/Album Name"

Example:

uv run beet --config ../groove-data/beets.yaml import -C -I \
  "../groove-data/library/Marvin Gaye/Here, My Dear"
  • -C – update files in place (don't move them).
  • -I – re-process even if already imported.
  • Add -t to confirm matches interactively (press A to apply, M for more candidates).
  • Add --from-scratch if every file was misnamed the same way and you want tags wiped before matching.

You can also cd into the album folder and use . as the path:

cd "../groove-data/library/Marvin Gaye/Here, My Dear"
uv run --directory "/path/to/groove" beet --config ../groove-data/beets.yaml import -C -I .

For many albums at once: uv run groove metadata retag-albums (try --dry-run or --artist "Name" --limit 1 first).

▶ Review

Files that beets couldn't auto-match land in inbox/review/. Accept a MusicBrainz candidate, or enter tags manually to re-import.

▶ Troubleshooting

Common issues

  • Download failed – click Retry in the queue, or supply a manual YouTube URL.
  • Wrong album match – run beet import -L "album:Name" in a terminal, or re-import the folder (see Library → Fix album track numbers).
  • Wrong tags / all files same name / bad track numbers – 
    uv run beet --config ../groove-data/beets.yaml import -C -I \
  "../groove-data/library/Artist Name/Album Name"
  • Wrong tags on one track – use the Edit button, or beet modify on the CLI.
  • Bulk “already in library” but files are bad – fix with the album import command above (don't re-queue the playlist). Or remove with beet remove -d 'album:Name' 'artist:Artist' then re-download.
  • Drive unplugged mid-download – worker pauses automatically; replug and it resumes.
  • Storage full – run groove doctor to see usage and get guidance.

Run groove doctor to diagnose any setup issues.
